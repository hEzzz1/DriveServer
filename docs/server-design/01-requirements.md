# 需求规格说明书（SRS）

## 1. 引言
### 1.1 目的
本文档用于定义“驾驶员疲劳与分心驾驶实时监测系统（服务器端）”的需求边界、核心功能与验收标准，作为后续设计、开发、测试和长期运维依据。

### 1.2 系统范围
- 包含：服务器接入、规则判定、告警闭环、统计分析、权限与审计。
- 不包含：边缘端硬件、摄像头驱动、端侧模型训练与推理实现。

### 1.3 术语
- 事件（Event）：上报的一次驾驶行为观测数据。
- 告警（Alert）：规则命中后创建的风险记录。
- 冷却时间（Cooldown）：同类告警短时间内不重复触发的抑制窗口。
- 闭环（Closure）：告警被确认、误报标注、关闭的完整处理过程。

## 2. 业务目标
1. 服务器能够实时接收并处理驾驶事件，触发风险告警。
2. 保障告警可追溯、可管理、可审计。
3. 提供趋势统计和排行分析，支撑运营决策与模型效果评估。
4. 具备可持续迭代能力，满足个人项目从 MVP 到稳定版本的演进。

## 3. 角色与使用场景
### 3.1 角色
- 管理员（ADMIN）：管理规则、账号、系统配置。
- 运营人员（OPERATOR）：处理告警、查看统计、导出报表。
- 观察者（VIEWER）：只读查看看板和历史数据。
- 设备（DEVICE）：仅可调用事件上报接口。

### 3.2 典型场景
1. 设备持续上报事件，系统实时判定并生成告警。
2. 运营人员在告警列表中确认、误报标注或关闭告警。
3. 管理员调整规则阈值，系统生效并记录审计日志。
4. 用户查看某车队近7天风险趋势与司机排行。

## 4. 功能需求
| 编号 | 功能 | 说明 | 优先级 |
|---|---|---|---|
| FR-001 | 设备鉴权 | 设备必须通过 `device_token` 调用上报接口 | P0 |
| FR-002 | 事件接收 | 接收单条事件，支持幂等去重 | P0 |
| FR-003 | 参数校验 | 时间、分数、ID等字段必须校验 | P0 |
| FR-004 | 异步入流 | 事件先写入 Redis Stream 再消费 | P0 |
| FR-005 | 实时判定 | 按阈值+持续时长判定风险 | P0 |
| FR-006 | 告警创建 | 命中规则后写 MySQL 告警主表 | P0 |
| FR-007 | 告警冷却 | 同车辆同规则在冷却窗口内不重复告警 | P0 |
| FR-008 | 告警推送 | 通过 WebSocket 实时推送给在线用户 | P1 |
| FR-009 | 告警查询 | 支持按时间、车辆、司机、状态分页查询 | P0 |
| FR-010 | 告警详情 | 查看触发条件、分值、处理日志 | P0 |
| FR-011 | 告警确认 | 运营人员可确认告警并记录操作人 | P0 |
| FR-012 | 误报标注 | 支持误报标注并写审计 | P1 |
| FR-013 | 告警关闭 | 支持关闭告警并记录原因 | P0 |
| FR-014 | 规则配置 | 支持阈值、时长、冷却时间配置 | P1 |
| FR-015 | 趋势统计 | 支持按时间窗查询风险趋势 | P1 |
| FR-016 | 排行统计 | 支持车辆/司机风险排行 | P1 |
| FR-017 | 权限管理 | JWT + RBAC 权限控制 | P0 |
| FR-018 | 操作审计 | 关键接口写审计日志 | P0 |
| FR-019 | 数据导出 | 告警列表和统计结果支持导出 | P2 |
| FR-020 | 健康检查 | 提供应用与依赖健康状态 | P0 |

## 5. 非功能需求
| 编号 | 指标 | 目标 |
|---|---|---|
| NFR-001 | 告警延迟 | 端到端 `P95 < 1s`（模拟数据） |
| NFR-002 | 吞吐能力 | `100~300 events/s` 持续10分钟无明显积压 |
| NFR-003 | 可用性 | 服务连续运行72小时无崩溃 |
| NFR-004 | 查询性能 | 常用报表接口平均响应 `< 2s` |
| NFR-005 | 安全性 | 鉴权、审计、最小权限、敏感数据脱敏 |
| NFR-006 | 可维护性 | 模块化代码结构、文档齐全、一键部署 |

## 6. 约束与假设
1. 边缘端行为由模拟器替代，数据格式稳定。
2. 开发环境采用 Docker Compose，生产环境可演进为 Kubernetes。
3. 实时通知以站内 WebSocket 为主，保留短信/IM 通道扩展能力。
4. 默认单地域部署，预留跨地域容灾的接口与数据策略。

## 7. 验收标准
1. 关键功能 FR-001~FR-013 全部通过。
2. 核心性能指标 NFR-001~NFR-004 达标。
3. 可交付完整文档、自动化测试、压测报告与运维手册。
