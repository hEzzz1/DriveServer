# 测试与验收文档

## 1. 测试目标
1. 验证功能正确性与业务闭环完整性。
2. 验证实时链路性能与稳定性。
3. 验证安全基线与异常恢复能力。

## 2. 测试环境
| 环境 | 说明 |
|---|---|
| Local | 开发联调，快速回归 |
| Stage | 预发布验证，执行压测与稳定性测试 |
| Prod-like | 近生产参数，执行验收测试 |

## 3. 测试范围
1. 功能测试：接口、规则、告警流转、统计查询。
2. 集成测试：MySQL/Redis/Influx 的协同行为。
3. 性能测试：高并发上报、查询压力、WebSocket推送。
4. 稳定性测试：长时运行、故障注入、异常重试。
5. 安全测试：鉴权、越权、注入、重放、防暴力破解。

## 5. 性能测试方案
### 5.1 上报压测
- 目标：验证 `100~300 events/s` 持续10分钟
- 观察项：
  - API成功率
  - P95响应时间
  - Stream积压增长趋势
  - 告警处理延迟

### 5.2 查询压测
- 目标：并发 50~100 用户查询告警与统计
- 观察项：
  - 平均响应时间
  - P95/P99 延迟
  - DB 慢查询数量
  - Redis 命中率

### 5.3 稳定性测试
- 目标：72小时持续运行
- 观察项：
  - 内存曲线是否稳定
  - 线程池是否异常增长
  - 死信数量是否可控

## 6. 验收指标
| 指标 | 目标值 |
|---|---|
| 告警链路延迟 | `P95 < 1s` |
| 事件接入吞吐 | `>=100 events/s` 稳定 |
| 核心接口成功率 | `>=99.5%` |
| 常用查询响应 | 平均 `<2s` |
| 系统稳定性 | 连续72小时无崩溃 |

## 7. 缺陷分级与准入
| 等级 | 定义 | 发布准入 |
|---|---|---|
| P0 | 核心功能不可用/数据错误 | 必须修复 |
| P1 | 主要功能受影响 | 原则上必须修复 |
| P2 | 次要功能问题 | 可评估延期 |
| P3 | 展示或文案问题 | 可后续修复 |

3. 性能测试报告。
4. 稳定性测试报告。
5. 缺陷清单与修复闭环记录。
