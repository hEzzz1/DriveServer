# 安全与合规设计文档

## 1. 安全目标
1. 防止未授权访问与越权操作。
2. 保障关键业务数据的机密性与完整性。
3. 提供可审计、可追踪、可追责能力。

## 2. 威胁模型（简版）
| 威胁 | 说明 | 对策 |
|---|---|---|
| 未授权调用 | 非法设备伪造上报 | 设备Token + IP白名单（可选） |
| 权限越权 | 普通用户调用管理接口 | JWT + RBAC 细粒度鉴权 |
| 数据篡改 | 传输中被篡改 | HTTPS/TLS |
| SQL注入 | 恶意构造输入 | 参数化查询 + 校验 |
| 重放攻击 | 重复上报旧事件 | eventId 幂等 + 时间窗校验 |
| 口令泄露 | 管理员弱密码 | 强密码策略 + 定期轮换 |

## 3. 认证与授权
1. 用户侧：用户名密码登录 + JWT。
2. 设备侧：`X-Device-Token` 鉴权。
3. 权限模型：
   - `ADMIN`：规则管理、用户管理、系统配置
   - `OPERATOR`：告警处理、查询统计
   - `VIEWER`：只读查看

## 4. 数据保护
### 4.1 传输安全
1. 外部访问强制 HTTPS。
2. 内网服务建议 TLS 或受控网络隔离。

### 4.2 存储安全
1. 密码只存 `hash`（BCrypt/Argon2）。
2. 敏感字段（如手机号）支持脱敏展示。
3. 备份文件加密存储并限制访问权限。

### 4.3 密钥与凭据管理
1. `JWT_SECRET`、DB密码不硬编码。
2. 使用环境变量或密钥管理服务。
3. 定期轮换密钥并保留切换窗口。

## 5. 审计策略
必须记录以下审计事件：
1. 登录成功与失败。
2. 规则新增、更新、启停。
3. 告警状态变更（确认、误报、关闭）。
4. 用户与角色权限变更。

审计字段建议：
- operator_id
- action
- module
- target_id
- detail_json
- ip
- created_at

## 6. 合规与隐私建议
1. 最小化数据采集原则，只采业务必需字段。
2. 数据保留周期明确并可配置。
3. 提供数据清理机制（到期自动删除）。
4. 导出数据应具备权限校验与脱敏策略。

## 7. 安全测试清单
1. 鉴权绕过测试。
2. 越权访问测试。
3. SQL 注入测试。
4. 弱口令与暴力破解测试。
5. 接口重放攻击测试。
6. 审计日志完整性检查。

## 8. 事件响应流程
1. 发现异常后立即升级告警级别。
2. 快速隔离受影响服务或账号。
3. 导出关键日志定位根因。
4. 执行修复、回滚、复盘并更新防护策略。
